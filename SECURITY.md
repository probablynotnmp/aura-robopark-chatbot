# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| Latest  | ✅        |
| Older   | ❌        |

> This project is actively maintained on its main branch. Only the latest release receives security patches.

## Reporting a Vulnerability

If you discover a security vulnerability in **Aura RoboPark Chatbot**, please **do not open a public GitHub issue**.

Instead, report it privately by emailing:

📧 **contact.prathyushmohan@gmail.com**

### What to include

- A clear description of the vulnerability
- Steps to reproduce the issue
- Potential impact (what an attacker could achieve)
- Any suggested fix, if you have one

### What to expect

- **Acknowledgement** within 48 hours of your report
- **Status update** within 7 days, confirming whether the vulnerability is accepted or declined
- If accepted, a patch will be prioritised and you will be credited in the release notes (unless you prefer to remain anonymous)
- If declined, you will receive a clear explanation of why it does not qualify as a security issue

### Scope

This policy covers the chatbot application code in this repository. Issues related to third-party libraries or services (e.g. the underlying LLM provider) should be reported directly to those vendors.

Thank you for helping keep this project secure.
